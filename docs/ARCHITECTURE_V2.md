# INNOCEAN Audience — 솔루션 구조도 v2 (2-Mode)

> 정의: 2개의 주요 작업 모드(AI Chat / Target Insight)와 공용 메뉴(Library / Projects / Settings)를 좌측 사이드바로 분리한다.
> 핵심 원칙: 같은 백엔드/데이터/캐시 공유. UI 진입점만 분리.

**특징**:
- 빌더는 숨김 (CEO 결정)
- 좌측 사이드바에서 모드 전환만 가능
- 동일 타겟에 후속 질문 가능 (대화형)
- 의도(intent)에 따라 차트 동적 선택 (1개~필요한 만큼, 줄이지 않음)
- 첫 질문 시 AI 패널 리서치 백그라운드 1회 (재활용)
- 답변마다 "타겟 인사이트로 자세히 보기" 링크 → 모드 전환

**의도 → 차트 매핑**:
```

values-insight     → mind(6) 중심

media-insight      → media(전체)

out-of-scope       → 안내 + 가능한 인사이트 1-2개
```

## 3. 데이터 모델 & 공유

### 3.1 공통 데이터 (두 모드가 공유)
- `state.currentCountry` — 현재 국가
- `state.filters` — 빌더 필터 (AI 채팅에서는 NLU가 채움, 사용자 직접 수정 불가)
- `state.audienceCache` — 5차원 실데이터 캐시
- `state.synthCache` — Gemini 합성 결과 (5차원 + 분포 필드)
- `state.panelCache` — 한국 30명 패널 결과
- `state.mediaCache` — 미디어 데이터

### 3.2 모드별 추가 데이터

**AI 채팅 전용** (신규):
```javascript
state.chatSessions = []     // 대화 세션 목록 (저장)
state.currentChatId = null  // 현재 활성 대화 id
state.chatMessages = []     // 현재 대화의 메시지 배열
                            // [{ role: 'user'|'assistant', content, charts, timestamp }]
state.chatTarget = null     // 현재 대화가 분석 중인 타겟 (filters + country)
```

**타겟 인사이트 전용**:
- 현재 9탭 관련 state 유지 (`currentTab`, `analysisRun`, `analyzedFilters` 등)

### 3.3 모드 전환 정책
- AI 채팅 → 타겟 인사이트: `state.filters`/`country` 그대로 넘어감. 자동 분석 시작 안 함 (사용자가 빌더에서 확인 후 [분석] 누름).
- 타겟 인사이트 → AI 채팅: 동일 타겟으로 "이 타겟에 대해 질문해 보세요" 입력창 활성화.
- "AI 채팅에서 타겟 인사이트로 보기" 버튼: 같은 필터로 타겟 인사이트 모드 + 자동 분석 → 요약 탭 진입.

## 4. 저장소 (Workspace) 소속

### 4.1 프로젝트 (Projects)
- 소속: **공용** (양 모드 공통)
- 담을 수 있는 것:
  - 분석된 타겟 (Target Insight 결과) ← 기존
  - AI 채팅 세션 (Chat Session) ← 신규
- 1 프로젝트 = N개 타겟 + N개 채팅 세션

### 4.2 라이브러리 (Library) — 3 섹션 구조
- **분석 히스토리**: 모든 분석된 타겟 (Target Insight 결과) — 자동 저장
- **즐겨찾기**: 사용자가 별표 단 타겟
- **프리셋**: 빌더 프리셋 (Target Insight 전용)
- **신규 추가: 채팅 히스토리** ← AI 채팅 세션 목록

> 라이브러리는 통합 보관소. 모드 구분 없이 "내가 분석한 모든 것"을 찾는 곳.

### 4.3 인박스 (Inbox) — 시스템 프로젝트
- 현재 그대로 유지 (자동 저장된 타겟이 임시 보관되는 프로젝트)
- 채팅 세션도 자동으로 인박스에 들어가는 것은 사용자 행위가 필요하므로 미적용 (선택 옵션)

## 5. URL 라우팅

```
/                       → AI 채팅 (기본 진입)
/chat                   → AI 채팅
/chat/:sessionId        → AI 채팅 특정 세션 열기
/insight                → 타겟 인사이트 (요약 탭)
/insight?tab=who        → 타겟 인사이트 인물상 탭
/insight?tab=who&country=KR → 인물상 + 한국
/projects               → 프로젝트 목록
/projects/:id           → 프로젝트 상세
/library                → 라이브러리
/library?section=chats  → 채팅 히스토리
/settings               → 설정
```

브라우저 히스토리 처리: 모드 전환 = pushState. 모드 안 탭/세션 변경 = pushState.

## 6. 백엔드 (변경 없음 + 신규 1개)

### 기존 (재사용)
- `GET /api/catalog` — 솔루션 메타
- `POST /api/ask` — NLU 자연어 파싱
- `GET /api/audience/:dim` — 차원별 실데이터
- `POST /api/audience/synthesize` — 5차원 합성 (Gemini)
- `POST /api/research/segment` — 한국 30명 패널
- `POST /api/insight/answer` — 답변 카드 생성
- `POST /api/interview/*` — 페르소나 인터뷰
- `GET/POST /api/library/*` — 라이브러리

### 신규
- `POST /api/chat/session` — 새 대화 세션 시작
- `GET /api/chat/session/:id` — 대화 세션 조회
- `POST /api/chat/session/:id/message` — 메시지 추가 (NLU+합성+답변 통합 워크플로우)
- `GET /api/chat/sessions` — 대화 세션 목록
- `DELETE /api/chat/session/:id` — 삭제

> 1단계는 클라이언트 localStorage로 저장하고 후속에서 서버 저장 (간단히 시작)

## 8. Edge Cases & 정책

| 케이스 | 처리 |
|--------|------|
| AI 채팅 첫 진입, 질문 없음 | 환영 화면 + 예시 카드 4개 (현재 홈과 동일) |
| 타겟 인사이트 첫 진입, 빌더 비어있음 | 국가 선택 화면 + 빌더 안내 |
| AI 채팅에서 범위 밖 질문 | 답변 카드를 노란색으로 + outOfScopeMessage |
| 모드 전환 시 데이터 보존 | filters/country 보존, 캐시 보존 |
| 동일 타겟 후속 질문 | 합성/패널 캐시 재사용 (재호출 안 함) |
| 새 대화 시작 | currentChatId 새로 생성, 대화 히스토리 클리어 |
| 대화 도중 다른 국가/필터로 변경하려면? | "이 질문은 다른 타겟이 필요합니다. 새 대화로 시작?" 안내 |
| 라이브러리에서 채팅 세션 클릭 | AI 채팅 모드로 전환 + 해당 세션 로드 |

## 9. 마이그레이션 단계

### Phase A (1차 — 1.5~2시간)
1. 좌측 사이드바 추가 (수직, 폭 220px)
2. 기존 홈 + 9탭 워크스페이스 → "타겟 인사이트" 모드 안으로 이동
3. AI 채팅 모드 신규 — 단일 질문/단일 답변 (현재 흐름 유지)
4. 모드 토글 + URL 라우팅

### Phase B (2차 — 1~2시간, 권장)
1. AI 채팅 대화형 (메시지 누적 + 후속 질문)
2. 의도별 차트 동적 표시
3. localStorage 채팅 세션 저장 + 라이브러리 채팅 히스토리 섹션

### Phase C (3차 — 추후)
1. 서버 저장 (PostgreSQL/Firestore)
2. 프로젝트에 채팅 세션 attach
3. 대화 공유/내보내기
